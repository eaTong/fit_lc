## 1. 产品概述

### 1.1 产品定位
FitLC 是一款 AI 健身记录 SaaS 系统，通过自然语言对话自动记录健身数据和身体围度，支持历史数据查询与趋势分析。

### 1.2 核心价值
- **零门槛记录**：用户无需手动填写表单，直接用自然语言描述训练内容
- **智能解析**：AI 自动解析训练动作、组数、次数、重量等参数
- **数据洞察**：自动汇总训练统计和围度变化趋势

### 1.3 目标用户
| 用户类型 | 描述 |
|---------|------|
| 个人健身爱好者 | 希望快速记录训练、追踪围度变化的普通用户 |
| 健身教练 | 管理客户训练数据（未来扩展） |
| 系统管理员 | 维护动作库、肌肉库等基础数据 |

## 3. 核心功能模块

### 3.1 AI对话记录 — ✅ 已实现

用户通过自然语言与AI对话，系统自动识别用户意图（记录训练/记录围度/查询数据）并执行相应操作。

**触发场景：**
- 记录训练 (save_workout)：`"深蹲100kg 5组每组8个"`
- 记录围度 (save_measurement)：`"今天胸围94，腰围78"`
- 查询训练 (query_workout)：`"这周跑了多少次？"`
- 查询围度 (query_measurement)：`"我的围度有什么变化？"`

**已实现的 Tools：**
- `saveWorkout` — 保存训练记录
- `saveMeasurement` — 保存围度记录
- `queryWorkout` — 查询训练历史
- `queryMeasurement` — 查询围度历史
- `generatePlan` — AI生成健身计划
- `adjustPlan` — 调整现有计划
- `analyzeExecution` — 分析执行数据
- `analyzeImage` — 分析用户上传的图片

### 3.2 训练动作库 — ✅ 已实现

结构化的训练动作数据库，支持按肌肉群、器械、难度筛选。

**动作属性：** name, category, equipment, difficulty, description, steps, safetyNotes, commonMistakes, exerciseType, variantType, tags, status, isVariant, parentId

**动作-肌肉关联：** exerciseId, muscleId, role (primary/secondary)

**审核流程：** AI或管理员创建 → status: `draft` → 管理员审核 → status: `published`

### 3.3 肌肉库 — ✅ 已实现

两层层级结构的肌肉数据库：肌肉群（Level 1）→ 主肌肉（Level 2）

```
胸部 (chest) → 胸大肌、胸小肌、前锯肌
背部 (back) → 背阔肌、中下斜方肌、大圆肌、小圆肌、竖脊肌
腿部 (legs) → 股四头肌、腘绳肌、臀大肌、小腿肌群
肩部 (shoulders) → 三角肌、肩袖肌群
手臂 (arms) → 肱二头肌、肱三头肌、前臂肌群
核心 (core) → 腹直肌、腹斜肌、腹横肌、下背肌群
```

### 3.4 历史记录管理 — ✅ 已实现

**训练记录 (Workout)：** id, userId, date, deletedAt
**训练动作 (WorkoutExercise)：** id, workoutId, name, sets, reps, weight, duration, distance
**围度记录 (BodyMeasurement)：** id, userId, date (DATETIME，支持同一天多次记录), deletedAt
**围度项目 (MeasurementItem)：** id, measurementId, bodyPart (支持 chest/waist/hips/biceps/thighs/calves/weight/bodyFat/other), value

**软删除：** 所有记录支持软删除（`deletedAt` 时间戳）

**同一天多次记录 (v1.9)：** 体重(weight)和体脂率(bodyFat)可以一天记录多次（如早晩各一次），通过时间戳区分。

### 3.5 数据趋势分析 — ✅ 已实现

- 围度趋势图：折线图展示胸/腰/臀/臂围度变化
- 训练统计：柱状图展示每周训练次数
- AI 智能总结（AIInsightSummary）：趋势页顶部AI总结
- 重点变化 Highlight（KeyChangesHighlight）：突出进步/退步最大项

### 3.6 健身计划 — ✅ 已实现

用户通过AI生成个性化训练计划，支持对话调整、可视化编辑、执行追踪。

**核心流程：** 生成计划 → 对话调整 → 执行打卡 → 进度分析

**AI Tools：** generate_plan, adjust_plan, analyze_execution, analyze_image

**计划状态：** draft / active / completed / paused

### 3.7 相册功能 — ✅ 已实现 (v1.7 更新)

用户相册功能，聊天中发送的图片自动同步到相册，支持按月份分组浏览和软删除。

**核心功能：**
- 聊天图片自动同步：用户发送的图片自动保存到相册
- 月份分组显示：所有照片按月份分组，最新月份在前
- 照片预览：点击可全屏预览
- 软删除：支持删除照片（设置 deletedAt）
- 公开URL：OSS 使用公开 URL，无需签名

**数据模型：**
- `album_photos` 表：id, userId, ossUrl, chatMessageId, createdAt, deletedAt

**围度页面左右对称部位 (v1.8)：**
- 左臂围/右臂围：同一行左右并排显示
- 左大腿围/右大腿围：同一行左右并排显示
- 左小腿围/右小腿围：同一行左右并排显示

**设置页面拆分（v1.7）：**
- /settings — 设置中心（基础信息、身体数据入口）
- /settings/profile — 头像、昵称编辑

### 4.5 Vision Preprocessor 图片预处理 — ✅ 已实现 (F-030)

用户发送图片时，通过 Zhipu AI 预处理图片分析，结果注入消息上下文供主 AI 使用。

**实现位置：** `backend/src/agents/plugins/visionPreprocessor.ts`

**流程：**
1. VisionPreprocessor 插件拦截含图片消息
2. 调用 Zhipu GLM-4V-Flash 分析图片
3. 将分析结果注入消息前缀
4. 主 AI (MiniMax) 处理纯文本对话

## 三、新增工具规划（8个）

### A. 训练辅助类（4个）

#### A1. 1RM 极限重量计算器 ⭐ 最高优先

**描述：** 输入训练重量和重复次数，推算1RM（一次最大重量），并给出不同训练目标对应的重量建议表。

**核心公式：**
- Epley: `1RM = w × (1 + r/30)`
- Brzycki: `1RM = w × (36/(37-r))`
- Lombardi: `1RM = w × r^0.10`
- 取多公式平均值

**输入：**
- 动作名称（可选，支持搜索）
- 重量 (kg)
- 重复次数

**输出：**
- 估算 1RM
- 训练强度表（力量 85-95% / 增肌 67-85% / 耐力 <67% 对应的重量和次数）
- 与历史 PR 对比（如有）

**数据打通：** PersonalRecord 表可自动填入该动作的历史最大重量

**UI 特性：** 滑块调整重量/次数，实时刷新结果

---

#### A2. RPE/RIR 重量换算器

**描述：** 基于当天的疲劳感知(RPE 1-10)，反推应使用的训练重量。

**核心公式：**
- `Weight = 1RM × (1 - (10 - RPE) × 0.03)` (简化版)
- RPE 10 = 1RM, RPE 9 = 97% 1RM, RPE 8 = 94% ...

**输入：**
- 目标 RPE (1-10 滑块)
- 1RM 或训练重量（可自动从 A1 或 PersonalRecord 获取）

**输出：**
- 建议重量
- RPE-重量对照表
- 预估可完成次数

**数据打通：** 可直接读取 A1 的 1RM 结果，或从 PersonalRecord 读取

---

#### A3. 杠铃片重计算器

**描述：** 输入目标总重量，计算杠铃两端各装哪些片。

**输入：**
- 目标重量 (kg)
- 杠铃重量（默认 20kg 空杆 / 可选 15kg 女杆）
- 可用片重（默认 25/20/15/10/5/2.5/1.25）

**输出：**
- 每端片重组合（从内到外排列）
- 图示（可视化杠铃片排列）

**UI 特性：** 可视化杠铃片排列图

---

#### A4. 热身配重计算器

**描述：** 输入正式组重量，自动生成递增热身组方案。

**输入：**
- 正式组重量 (kg)
- 热身组数（默认 3-4 组）

**输出：**
- 热身组方案（如：空杆×10 → 50%×8 → 70%×5 → 85%×3）
- 每组的杠铃片排列（复用 A3）

**数据打通：** 可从 A1 的 1RM 结果或 PersonalRecord 推算

---

### B. 身体指标类（3个）

#### B1. 体脂率计算器

**描述：** 海军皮尺法（US Navy），通过腰围、颈围、身高估算体脂率。

**核心公式：**
- 男性: `BF% = 86.010 × log10(腹围-颈围) - 70.041 × log10(身高) + 36.76`
- 女性: `BF% = 163.205 × log10(腰围+臀围-颈围) - 97.684 × log10(身高) - 78.387`

**输入：**
- 性别
- 身高（可从 UserProfile 自动填入）
- 腰围、颈围（女性需臀围）
- 单位切换（cm/inch）

**输出：**
- 体脂率 (%)
- 体脂分类区间（偏低/正常/偏高/肥胖）
- 与上次测量对比

**数据打通：** MeasurementItem 中腰/颈/臀围数据可自动读入；BodyMetrics 中的 bodyFat 可做历史对比

---

#### B2. TDEE / BMR 计算器

**描述：** 计算每日总能量消耗(TDEE)和基础代谢率(BMR)。

**核心公式：**
- Mifflin-St Jeor (推荐): 男性 `BMR = 10×体重 + 6.25×身高 - 5×年龄 + 5`；女性 `... - 161`
- TDEE = BMR × 活动系数 (1.2/1.375/1.55/1.725/1.9)

**输入：**
- 性别、年龄
- 身高（可从 UserProfile 自动填入）
- 体重（可从 BodyMetrics 最新记录自动填入）
- 活动水平（5级选择）

**输出：**
- BMR (kcal)
- TDEE (kcal)
- 不同目标的每日热量（减脂 -500 / 维持 / 增肌 +300）

**数据打通：** UserProfile + BodyMetrics 自动预填

---

#### B3. BMI 计算器

**描述：** 计算身体质量指数，最简单的健康指标计算器。

**公式：** `BMI = 体重(kg) / 身高(m)²`

**输入：**
- 身高（可从 UserProfile 自动填入）
- 体重（可从 BodyMetrics 自动填入）

**输出：**
- BMI 数值
- 分类（偏瘦/正常/超重/肥胖）
- 健康体重范围

**数据打通：** UserProfile + BodyMetrics 全自动预填，用户只需点一下

---

### C. 营养计算类（2个，均为 TDEE 的延伸）

#### C1. 宏量素计算器

**描述：** 基于 TDEE 和目标，分配蛋白质/碳水/脂肪的每日克数。

**输入：**
- TDEE（可从 B2 结果自动获取）
- 目标（减脂/维持/增肌）
- 饮食偏好（标准/低碳/高蛋白）

**输出：**
- 每日蛋白质(g) / 碳水(g) / 脂肪(g)
- 每餐分配建议
- 每克对应的食物量参考（如 30g 蛋白 ≈ 100g 鸡胸肉）

**数据打通：** 直接读取 B2 的 TDEE 结果

---

#### C2. 饮水量计算器

**描述：** 根据体重和训练强度，计算每日推荐饮水量。

**公式：**
- 基础: `体重(kg) × 35ml`
- 训练日: `+ 训练时长(min) × 10ml`

**输入：**
- 体重（可自动填入）
- 训练频率
- 气候因素（高温+500ml）

**输出：**
- 每日推荐饮水量 (ml)
- 分时段饮水建议

**数据打通：** BodyMetrics 体重自动填入

## 四、实现优先级

### 第一批（核心刚需，用户价值最高）

| 优先级 | 工具 | 理由 |
|--------|------|------|
| P0 | A1. 1RM 极限重量计算器 | 健身计算器之王，4/4竞品必备，可接 PersonalRecord |
| P0 | B2. TDEE/BMR 计算器 | 减脂增肌第一问，可接 BodyMetrics |
| P0 | B1. 体脂率计算器 | 围度数据已有，海军法只需3个输入 |

### 第二批（延伸价值，工具间联动）

| 优先级 | 工具 | 理由 |
|--------|------|------|
| P1 | A2. RPE/RIR 重量换算器 | 需 1RM 数据，与 A1 联动 |
| P1 | C1. 宏量素计算器 | 需 TDEE 数据，与 B2 联动 |
| P1 | A3. 杠铃片重计算器 | 独立可用，但与 A1/A4 组合更强 |

### 第三批（锦上添花）

| 优先级 | 工具 | 理由 |
|--------|------|------|
| P2 | B3. BMI 计算器 | 最简单，但专业用户不关注 |
| P2 | A4. 热身配重计算器 | 依赖 A3，使用频率较低 |
| P2 | C2. 饮水量计算器 | 独立计算，价值较低 |

## 五、技术架构

### 前端实现方案

**Web 端 + 小程序端同步实现**，核心计算逻辑提取为独立模块。

```
frontend/src/
  utils/
    fitnessCalculators.ts    # 纯函数计算库（所有公式）
  pages/
    Tools.tsx                 # 工具列表页（Web 新增）
    tools/
      OneRMCalculator.tsx     # 1RM 计算器
      RPECalculator.tsx       # RPE 换算器
      PlateCalculator.tsx     # 片重计算器
      WarmupCalculator.tsx    # 热身配重
      BodyfatCalculator.tsx   # 体脂率
      TDEECalculator.tsx      # TDEE/BMR
      BMICalculator.tsx       # BMI
      MacroCalculator.tsx     # 宏量素
      WaterCalculator.tsx     # 饮水量

fitlc-mini/
  pages/
    tools/
      tools.wxml/js/wxss      # 更新工具列表
    tools-detail-onerm/       # 1RM 计算器
    tools-detail-rpe/         # RPE 换算器
    tools-detail-plate/       # 片重计算器
    tools-detail-bodyfat/     # 体脂率
    tools-detail-tdee/        # TDEE
    ...
```

### 后端 API（可选）

大部分计算器为纯前端计算，不需要后端。但以下数据预填需要 API：

| API | 用途 | 已有？ |
|-----|------|--------|
| `GET /api/records/pr/:exerciseName` | 获取动作 PR | 需新增 |
| `GET /api/profiles/me` | 获取身高/性别 | 已有 |
| `GET /api/body-metrics/latest` | 获取最新体重/体脂 | 需新增 |
| `GET /api/measurements/latest` | 获取最新围度 | 需新增 |

### 工具间数据联动

```
PersonalRecord ──→ A1(1RM) ──→ A2(RPE) ──→ A4(热身)
                                    │              │
                                    └──→ A3(片重) ←─┘

BodyMetrics ──→ B2(TDEE) ──→ C1(宏量素)
     │              └──→ B3(BMI)
UserProfile(身高/性别) ──→ B1(体脂率)
MeasurementItem(围度) ──→ B1(体脂率)
```

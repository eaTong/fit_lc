# 围度记录内嵌编辑功能设计

**日期：** 2026-05-05
**状态：** 已批准
**类型：** 小程序功能优化

---

## 1. 需求背景

当前围度记录页面（measurements）仅展示已录入的数据，用户无法直观看到哪些部位未录入并快速补录。本功能优化为：
- 所有围度部位均展示出来（无论是否录入）
- 点击未录入部位可直接快捷录入

---

## 2. UI 布局

### 2.1 整体结构

```
+-------------------+-------------------+
|    体重 (kg)      |      胸围 (cm)     |
+-------------------+-------------------+
|    腰围 (cm)      |      臀围 (cm)     |
+-------------------+-------------------+
|   左臂围  |  右臂围 |   左大腿  | 右大腿 |
+-------------------+-------------------+
|   左小腿  |  右小腿 |              |
+-------------------+-------------------+
```

### 2.2 部位分类

| 类型 | 部位 | 布局 |
|------|------|------|
| 独立部位 | 体重、胸围、腰围、臀围 | 单独占格 |
| 对称部位 | 臂围、大腿围、小腿围 | 左右并排占格 |
| 计算得出 | 体脂率 | 单独占格（如有，计算来源不同） |

### 2.3 单元格状态

| 状态 | 视觉表现 |
|------|----------|
| 已录入 | 实线边框，显示数值 + 单位 |
| 未录入 | 虚线边框，显示"点击录入"或部位名 |
| 对称部位单边录入 | 已录边显示数值，另一边虚线"点击录入" |

---

## 3. 交互流程

### 3.1 录入流程

1. 用户点击空部位格（虚线边框）
2. 弹出内置输入框（数字键盘），显示部位名和单位
3. 输入数值，点击确认
4. 调用保存 API
5. 成功后更新格内显示 + 高亮动画（背景色从 #1A1A1A 闪动至 #FF4500 再渐变回 #1A1A1A，200ms）
6. 失败时 toast 提示"保存失败"

### 3.2 取消流程

1. 点击遮罩层或取消按钮
2. 输入框关闭，无变化

## 4. 数据结构

```javascript
// 围度数据模型
{
  id: string,
  date: '2026-05-05',
  weight: 75.5,        // 体重(kg)
  chest: 100,          // 胸围(cm)
  waist: 80,           // 腰围(cm)
  hips: 95,            // 臀围(cm)
  biceps_l: 35,        // 左臂围(cm)
  biceps_r: null,      // 右臂围 - 未录入
  thigh_l: 55,         // 左大腿围(cm)
  thigh_r: null,       // 右大腿围 - 未录入
  calf_l: 38,          // 左小腿围(cm)
  calf_r: null,        // 右小腿围 - 未录入
  bodyFat: 15          // 体脂率(%)
}
```

## 5. 技术实现

### 5.1 涉及文件

- `fitlc-mini/packageB/pages/measurements/index.wxml` — 主页面模板
- `fitlc-mini/packageB/pages/measurements/index.js` — 页面逻辑
- `fitlc-mini/packageB/pages/measurements/index.wxss` — 样式

### 5.2 API 调用

- **保存围度：** `POST /api/records/measurement`
  - 请求体：`{ date, measurements: [{ body_part, value }] }`
  - 响应体：`{ id, date, ... }`

### 5.3 状态管理

- 前端 store 需支持单个部位更新
- `updateMeasurement(bodyPart, value)` — 更新指定部位

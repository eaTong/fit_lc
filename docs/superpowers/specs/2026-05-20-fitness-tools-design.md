# 健身工具页面设计方案

## 概述

在微信小程序中新增"工具"TabBar页面，提供三个健身辅助工具：10*10训练计时器、计数器、通用计时器。训练数据保存为正式训练记录。

## 入口

- **位置**：TabBar 第2位（对话之后，动作库之前）
- **Tab名称**：工具
- **Tab图标**：assets/tabbar/tools.png / tools-active.png

## 页面结构

页面 `pages/tools/tools` 包含三个工具 Tab：

| Tab | 名称 | 数据保存 |
|-----|------|----------|
| tab1 | 10*10 | Workout |
| tab2 | 计数 | Workout |
| tab3 | 计时 | 不保存 |

---

## Tab 1：10*10 训练

### 配置阶段

| 字段 | 默认值 | 说明 |
|------|--------|------|
| 动作名称 | 俯卧撑 | 可选：俯卧撑、深蹲、自定义（文本输入） |
| 组间休息 | 自动（60s） | 根据预计总时长自动建议，可手动调整 |
| 开始按钮 | — | 开始训练 |

**自动建议休息时长逻辑：**

根据用户选择的动作和目标，计算预计总锻炼时长（不含休息），自动建议合适的组间歇：

| 预计总时长 | 建议休息 | 说明 |
|------------|----------|------|
| < 10 分钟 | 45 秒 | 短时高强度 |
| 10-20 分钟 | 60 秒 | 中等时长 |
| 20-30 分钟 | 90 秒 | 标准 10*10 |
| > 30 分钟 | 120 秒 | 长时恢复较慢 |

预计总时长计算：`(10组 × 10次 × 预计每次用时 + 9次 × 组间歇时长) / 60` 分钟
- 俯卧撑/深蹲：预计每次 3 秒
- 基准场景：10×10 + 60s 间歇 ≈ 18 分钟总时长

### 训练阶段

**布局（从上到下）：**
1. 进度指示："第 3 组 / 10 组"
2. 当前组进度："第 7 次 / 10 次"（大字号）
3. 大按钮："完成一组"（每点一次完成当前组次）

**状态逻辑：**
- 初始：组=1，次=0
- 每点击"完成一组"：次归零，组+1，进入休息倒计时
- 用户按自己节奏做，做完一组就点一次

### 休息阶段

**布局：**
1. 提示文字："休息一下"
2. 大字号倒计时（60-1 递减）
3. "跳过休息"按钮

**状态逻辑：**
- 倒计时结束 → 震动+提示音 → 自动进入下一组
- 用户可点击"跳过休息"提前开始

### 完成阶段

- 显示训练汇总："完成 10 组，共 100 次"
- 弹出保存确认对话框
- 确认后调用保存接口

**中途退出：**
- 用户可随时点击"放弃训练"
- 弹出确认："确定要放弃当前训练吗？数据将不会保存"
- 若已完成至少 1 组，提示："已完成了 X 组，是否保存当前进度？"

### 保存数据格式

```typescript
{
  date: Date.now(),
  exercises: [{
    name: "俯卧撑",
    sets: 10,       // 完成组数
    reps: 10        // 每组固定10次
  }],
  totalDuration: number // 秒，从开始到结束的总时长
}
```

## Tab 2：计数器

### 配置阶段

| 字段 | 默认值 |
|------|--------|
| 动作类型 | 俯卧撑 | 可选：俯卧撑、深蹲、自定义（文本输入） |
| 目标次数 | 20 | 可调 |

### 计数阶段

**布局：**
1. 动作名称 + 目标显示："俯卧撑 / 目标 20 次"
2. 当前计数（大字号）："15"
3. 进度条：已完成 / 目标
4. 大按钮："计数 +1"

**状态逻辑：**
- 每次点击：计数 +1
- 达到目标：震动+提示音，显示"恭喜完成！"
- 可继续计数（超过目标后累计显示）

### 保存

- "保存记录"按钮（训练结束后可见）
- 保存格式：`sets: 1, reps: 实际完成总数`
- 例如：完成 47 次俯卧撑 → `{ name: "俯卧撑", sets: 1, reps: 47 }`

---

## Tab 3：计时器

### 配置阶段

**快速选项按钮：** 30s / 60s / 90s / 120s / 自定义

**自定义：** 点击后弹出数字键盘输入秒数

### 计时阶段

**布局：**
1. 大字号倒计时显示（如 "01:30"）
2. 进度环（可选）
3. "暂停/继续" 按钮
4. "停止" 按钮

**状态逻辑：**
- 倒计时结束 → 震动+提示音 → 显示"时间到！"
- 点击"停止" → 返回配置阶段

**不保存数据**

## 技术实现

### 页面文件

```
pages/tools/
├── tools.js
├── tools.wxml
├── tools.wxss
└── tools.json
```

### 样式

- 背景色：#0A0A0A
- 卡片色：#1A1A1A
- 强调色：#FF4500
- 文字色：#FFFFFF / #888888
- 按钮：2px solid #FF4500，圆角 4px
- 大按钮高度：120rpx，便于点击

### API

保存训练记录调用现有接口：
```
POST /api/records/workout
```

### 状态管理

工具页面为独立页面，不与 chatStore 共享状态。使用页面自身 data 管理工具状态。

### 提示音

- 使用 wx.vibrateLong() 震动
- 提示音使用 wx.playBackgroundAudio() 或简单系统提示

# F-001 保存成功视觉卡片设计

**创建时间：** 2026-04-26
**功能ID：** F-001
**状态：** 设计完成

---

## 一、概述

**功能名称：** 保存成功视觉卡片
**功能描述：** 训练记录保存成功后，在聊天消息中插入视觉卡片展示保存的数据，支持修改和查看详情操作。

**优先级：** P0（L1 基础功能层）
**依赖关系：** F-004 一键记录模式的展示载体

---

## 二、视觉风格

**风格：** 深色毛玻璃卡片
- 背景：rgba(255,255,255,0.08) 半透明
- 边框：1px solid rgba(255,255,255,0.15)
- 模糊：backdrop-filter: blur(12px)
- 圆角：16px

---

## 三、卡片结构

```
┌─────────────────────────────────────┐
│ 🎉 训练记录已保存          [成功]   │  ← 头部：图标 + 状态
├─────────────────────────────────────┤
│           深蹲                        │  ← 动作名称
│  ┌─────┐ ┌─────┐ ┌─────┐           │
│  │100kg│ │ 5组 │ │ 8个 │           │  ← 关键数值
│  └─────┘ └─────┘ └─────┘           │
├─────────────────────────────────────┤
│ [腿部] [复合动作] 今天 14:32         │  ← 标签
├─────────────────────────────────────┤
│   [修改]        [查看详情]           │  ← 操作按钮
└─────────────────────────────────────┘
```

---

## 四、展示内容

| 区块 | 内容 | 说明 |
|------|------|------|
| 头部 | 图标 + "训练记录已保存" + 状态标签 | 成功状态绿色标签 |
| 动作详情 | 动作名称 + 重量/组数/次数/时长/距离 | 根据实际数据动态展示 |
| 标签 | 肌肉群、动作类型、时间 | 可选展示 |
| 操作 | 修改按钮、查看详情按钮 |  |

---

## 五、操作行为

### 5.1 修改按钮

- **行为：** 弹窗编辑本次保存的数据
- **编辑后：** 覆盖原记录，重新保存
- **交互：** 点击 → 弹出编辑弹窗 → 修改数据 → 确认保存

### 5.2 查看详情按钮

- **行为：** 跳转到记录详情页
- **目标：** /records/workout/:id 或类似详情页

## 六、技术实现

### 6.1 组件设计

```tsx
// components/SaveSuccessCard.tsx
interface SaveSuccessCardProps {
  savedData: {
    type: 'workout';
    id: number;
    exercises: Exercise[];
    timestamp: string;
  };
  onEdit: (id: number, data: Exercise[]) => void;
  onViewDetail: (id: number) => void;
}
```

### 6.2 状态管理

| 状态 | 说明 |
|------|------|
| default | 展示卡片内容 |
| editing | 弹窗编辑中 |
| saving | 保存中 |
| error | 保存失败 |

### 6.3 编辑弹窗

- 展示当前保存的数据
- 支持修改动作名称、重量、组数、次数等
- 确认后调用更新接口

## 七、接口契约

### 7.1 触发条件

当 `/api/chat` 响应中 `savedData` 存在且 `savedData.type === 'workout'` 时，前端渲染视觉卡片。

### 7.2 数据结构

```typescript
interface SavedWorkoutData {
  type: 'workout';
  id: number;
  exercises: {
    name: string;
    sets?: { weight: number; reps: number }[];
    duration?: number;
    distance?: number;
  }[];
  timestamp: string;
}
```

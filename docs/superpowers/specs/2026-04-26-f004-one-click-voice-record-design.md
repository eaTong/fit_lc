# F-004 一键记录模式设计

**创建时间：** 2026-04-26
**功能ID：** F-004
**状态：** 设计完成

---

## 一、概述

**功能名称：** 一键记录模式
**功能描述：** 在现有聊天界面底部增加"按住说话"按钮，语音转文字后自动发送到 /api/chat，AI 正常回复，保存成功时插入 F-001 视觉卡片。

**优先级：** P0（L1 基础功能层）
**依赖关系：** 无依赖，可最先开发

---

## 二、核心交互

### 2.1 主要流程

```
用户按住麦克风 → 录音 → 松开 → 语音转文字 → 自动发送至 /api/chat
                                                            ↓
                                              Agent 解析 → save_workout 工具
                                                            ↓
                                              返回 reply + savedData
                                                            ↓
                                              前端检测 savedData → 显示视觉卡片 + 播放音效
```

### 2.2 支持的交互节奏

**一句完成（核心实现）：**
- 语音输入 → 松开 → 直接保存 → 显示视觉卡片 → 播放成功音效
- 适用场景：简单明确的训练记录（如"深蹲100公斤5组8个"）

**对话式（后续扩展）：**
- 说"今天练了" → AI 追问"练了什么？" → 用户继续说 → AI 保存

**连续记录（后续扩展）：**
- 可多次说动作，最后说"保存"统一确认

---

## 三、技术架构

### 3.1 数据流

```
麦克风 → 语音转文字(Web Speech API) → /api/chat → LangChain Agent → save_workout 工具
                                                                        ↓
                                                              reply + savedData
                                                                        ↓
                                                    前端判断 savedData → 显示视觉卡片
```

### 3.2 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端语音 | Web Speech API / 第三方SDK | 语音转文字 |
| 后端接口 | 复用 /api/chat | 无需新增接口 |
| AI解析 | 复用现有 Agent | LangChain + save_workout tool |
| 视觉反馈 | F-001 视觉卡片组件 | 已规划功能 |

---

## 四、状态管理

| 状态 | UI表现 | 说明 |
|------|--------|------|
| idle | 麦克风按钮可点击，文字"按住说话" | 空闲状态 |
| recording | 按钮变红+波形动画，文字"松开结束" | 录音中 |
| processing | 按钮禁用，显示加载动画 | 解析中 |
| success | 显示视觉卡片，播放成功音效 | 成功保存 |
| partial | 显示部分数据，标记"待补充" | 部分成功 |
| failed | 显示错误提示，可重试 | 保存失败 |

---

## 五、接口契约

### 5.1 请求（复用 /api/chat）

```json
POST /api/chat/message
{
  "message": "深蹲100公斤5组8个",
  "historyMessages": []
}
```

### 5.2 响应

```json
{
  "reply": "已记录你的深蹲训练 💪\n\n100kg × 5组 × 8个",
  "savedData": {
    "type": "workout",
    "id": 123,
    "exercises": [{...}]
  }
}
```

**判断逻辑：** 当 `savedData` 存在且 `savedData.type === 'workout'` 时，前端显示 F-001 视觉卡片。

---

## 六、解析失败处理

### 6.1 部分可识别

- 保存能解析的部分（如只识别出"深蹲"但缺少重量）
- 数据标记为"待补充"状态
- 显示可编辑卡片，用户可手动补充

### 6.2 完全无法识别

- 不保存任何数据
- 显示"没听清，请再说一次"
- 提供重试按钮

---

## 七、前端实现要点

1. **语音组件：** 使用 Web Speech API 或第三方 SDK（推荐讯飞/腾讯语音）
2. **状态管理：** 使用 React useState 管理 idle/recording/processing/success/partial/failed
3. **自动发送：** 语音转文字完成后自动填充输入框并提交
4. **视觉卡片：** 复用 F-001 的视觉卡片组件，通过 savedData 驱动显示
5. **音效反馈：** 成功时播放简短音效（可使用 Web Audio API）

---

## 八、后端改动

**无新增接口。** 复用现有 /api/chat 接口，savedData 由 Agent 返回时自动包含。

# 聊天页面输入框设计

## 1. 需求概述

为 FitLC 小程序聊天页面重新设计底部输入框组件，支持语音/文字输入模式切换。

**核心功能：**
- 左侧语音/文字切换按钮，图标随状态变化
- 语音模式：按住说话，松开发送
- 文字模式：多行输入框
- 图片上传按钮始终显示在发送键左侧

---

## 2. 布局结构

```
┌─────────────────────────────────────────────┐
│ [🎤/⌨️] [   多行输入框/语音状态   ] [📷] [➤] │
└─────────────────────────────────────────────┘
```

**组件排列（左→右）：**
| 组件 | 宽度 | 说明 |
|------|------|------|
| 语音/文字切换按钮 | 64rpx | 图标按钮，最左侧 |
| 输入区域 | flex: 1 | 自适应宽度 |
| 图片上传按钮 | 64rpx | 始终显示 |
| 发送按钮 | 64rpx | 最右侧 |

---

## 3. 状态与交互

### 3.1 输入模式

| 状态 | 切换按钮图标 | 输入区显示 | 说明 |
|------|------------|-----------|------|
| 文字模式 | 🎤 (麦克风) | 多行输入框，placeholder="输入训练内容..." | 正常输入 |
| 语音模式 | ⌨️ (键盘) | 显示"按住说话"文字提示 | 按下时变为"松开发送" |

### 3.2 语音录制交互

| 事件 | 行为 |
|------|------|
| `bindtouchstart` | 开始录音，输入区显示"松开发送" |
| `bindtouchend` | 停止录音，发送语音消息 |
| 超时 60 秒 | 自动停止并发送 |

### 3.3 切换逻辑

- 点击切换按钮在语音/文字模式间切换
- 切换时重置录音状态（如有）
- 模式状态可持久化到本地存储

---

## 4. 视觉规格

### 4.1 输入框容器
- 背景：`var(--color-bg-tertiary)`
- 圆角：40rpx
- 内边距：8rpx 16rpx
- 最小高度：80rpx
- 最大高度：200rpx（auto-expand）

### 4.2 按钮规格
- 尺寸：64rpx × 64rpx
- 图标大小：36rpx
- 左边距：8rpx

### 4.3 语音状态样式
- 非录音态：placeholder "按住说话..."，`var(--color-text-secondary)`
- 录音中：文字 "松开发送"，`var(--color-accent-red)`

## 一、概述

**功能名称：** 你可能想提示
**功能描述：** 用户打开应用时，在顶部展示个性化提示（今日建议或欢迎回来），用户可关闭，无操作按钮。

**优先级：** P0（L1 基础功能层）
**依赖关系：** 无依赖

---

## 二、触发时机

**触发：** 用户打开应用时（首次打开或从后台唤醒）

**判断逻辑：**
1. 检查是否有今日建议
2. 检查距离上次训练时间
3. 显示对应类型的提示

---

## 三、提示类型

### 3.1 今日建议

**条件：** 有可用的训练建议时显示

**内容示例：**
```
💡 今日建议：腿部训练
```

**逻辑：**
- 基于用户的训练计划（如果有）
- 或基于肌肉群轮换规律
- 或基于季节/天气推荐

### 3.2 欢迎回来

**条件：** 距离上次训练超过 2 天时显示

**内容示例：**
```
👋 欢迎回来，距离上次训练已 3 天
```

**逻辑：**
- 计算 deleted_at 为空的最大训练日期
- 超过阈值（2天）显示欢迎提示
- 沉默周期：显示一次后 24 小时内不重复显示

## 五、交互行为

### 5.1 关闭行为

- 点击 × 关闭横幅
- 关闭后当天不再显示
- 使用 localStorage 记录关闭时间

### 5.2 展示优先级

1. 今日建议优先于欢迎回来
2. 只有一个提示，不能同时显示多个

### 5.3 频率控制

- 同一提示类型，24 小时内不重复展示
- 使用 localStorage 记录：`lastShown_${type}_${date}`

## 六、技术实现

### 6.1 数据来源

- 训练计划数据（plans 表）
- 历史训练记录（workouts 表）
- 用户设置偏好

### 6.2 判断逻辑

```javascript
// 伪代码
function shouldShowTip() {
  // 检查是否有今日建议
  const todaySuggestion = getTodaySuggestion();
  if (todaySuggestion) {
    return { type: 'suggestion', content: todaySuggestion };
  }

  // 检查距离上次训练
  const daysSinceLastWorkout = getDaysSinceLastWorkout();
  if (daysSinceLastWorkout >= 2) {
    return { type: 'welcome', content: `距离上次训练已 ${daysSinceLastWorkout} 天` };
  }

  return null; // 不显示提示
}
```

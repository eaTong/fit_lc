## 一、概述

**功能名称：** 智能总结先行
**功能描述：** 在趋势页顶部展示 AI 生成的本周/本月核心变化总结，用户可点击展开查看详细数据、趋势图表和历史对比。

**优先级：** P0（L1 基础功能层）
**依赖关系：** 数据基础（L2 的 F-002/F-003）提供原始数据

---

## 二、展示位置

**趋势页顶部：**
```
┌─────────────────────────────────────────────┐
│  📊 AI 智能总结                              │
│                                             │
│  本周训练4次，深蹲重量提升至100kg，较上周    │
│  增加5kg。总体训练量稳定。                  │
│                                             │
│  [查看详情 ▼]                               │
└─────────────────────────────────────────────┘
```

---

## 三、AI 总结内容

### 3.1 核心变化（默认展示）

**包含：**
- 本周训练次数
- 关键指标变化（如重量提升）
- 总体趋势评价（稳定/进步/需注意）

**格式：** 一段话，2-3 句

### 3.2 展开后内容（点击查看详情）

**详细数据：**
- 各动作的重量/次数变化
- 训练时长变化
- 围度变化（如有）

**趋势图表：**
- 折线图展示时间变化
- 柱状图展示训练量对比

**历史对比：**
- 与上周对比（上周 vs 本周）
- 与上月对比（上月 vs 本月）
- 与去年同期对比（如有数据）

---

## 四、折叠/展开交互

### 4.1 默认状态

- 展示简洁总结（一段话）
- 显示"查看详情"按钮

### 4.2 展开状态

- 显示详细数据区域
- 按钮变为"收起详情▲"
- 支持收起

---

## 五、技术实现

### 5.1 数据来源

- 训练记录数据（workouts 表）
- 围度数据（body_measurements 表）
- 累计统计数据（D-004）

### 5.2 AI 生成

调用现有 Agent 或单独的服务生成总结：
```javascript
// 伪代码
const summary = await aiService.generateSummary(userId, period);
// period: 'week' | 'month'
```

### 5.3 缓存策略

- 首次加载时生成
- 缓存 N 小时（避免频繁调用）
- 新增记录时更新缓存

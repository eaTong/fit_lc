## 1. 测试概述

本文档定义了 FitLC 系统的 E2E 测试规范，涵盖所有 P0 已实现功能的测试场景。

| 肌肉库 | muscles.spec.ts | 4 |

| 管理员-肌肉 | admin-muscles.spec.ts | 5 |

### 1.2 测试标识规范

```
[模块]-[序号]: [测试描述] @[标签]
```

示例：
- `CHAT-001: 记录训练-跑步 @chat`
- `ACH-002: 获取用户徽章 @achievements @badges`

## 3. 测试数据要求

### 3.1 测试用户

| 用户名 | 邮箱 | 密码 | 角色 | 用途 |
|--------|------|------|------|------|
| Test User | test@example.com | Test123456 | normal | 常规E2E测试 |
| Admin User | admin@example.com | Admin123456 | admin | 管理员功能测试 |

### 3.2 测试数据清理

每个测试用例执行前应确保：
1. 使用独立的测试用户或清理上次测试数据
2. 测试间无依赖关系
3. 使用 `test.beforeEach` 进行登录 setup

## 4. 测试执行

### 4.1 本地执行

```bash
cd frontend
npm run test:e2e
```

### 4.2 CI执行

```bash
export CI=true
npm run test:e2e
```

### 4.3 仅运行特定模块

```bash
npx playwright test --grep "@chat"
npx playwright test --grep "@achievements"
npx playwright test --grep "@celebration"
```
